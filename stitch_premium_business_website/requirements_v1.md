# 个人网站 V1 需求文档

版本：V1.0
日期：2026-04-17
定位：个人展示站 + AI 工具导航 + 博客 + 项目作品集

## 1. 项目背景

本项目用于建设一个个人网站，核心目的是向外界展示个人能力、兴趣方向、项目成果和对 AI 工具的长期整理与实践。

网站第一版不追求复杂后台和社区化功能，重点是快速上线、内容可持续维护、页面体验清晰，并为后续扩展搜索、订阅、工具评分、评论、用户投稿等功能预留空间。

## 2. 建站目标

### 2.1 核心目标

1. 建立个人线上名片，让访问者快速了解“我是谁、我会什么、我在做什么”。
2. 提供一个结构化的 AI 工具导航页，沉淀工具分类、使用场景和个人推荐。
3. 发布个人博客，用于记录技术、AI、项目、学习和经验总结。
4. 展示个人项目，通过项目详情页体现解决问题的能力。
5. 保持技术栈轻量，方便长期更新和部署。

### 2.2 非目标

V1 不做以下内容：

1. 用户注册与登录。
2. 复杂后台管理系统。
3. 付费会员体系。
4. 多人投稿审核系统。
5. 动态数据库驱动的工具评分系统。
6. 实时同步 AI 工具价格、模型能力或官网状态。

以上能力可作为后续版本规划。

## 3. 目标用户

### 3.1 普通访问者

想快速了解站主是谁、做过什么、关注什么方向。

### 3.2 技术同行

关注博客、项目实现、技术栈、开源代码和实践经验。

### 3.3 AI 工具使用者

希望找到适合写作、编程、图片、视频、效率、办公、搜索等场景的 AI 工具。

### 3.4 潜在合作方

希望通过项目、文章和联系方式判断是否可以合作。

## 4. 内容定位

网站不是普通链接聚合站，而是带有个人判断力的 AI 资源库和作品展示站。

核心差异化来自：

1. 工具筛选理由。
2. 真实使用经验。
3. 项目复盘。
4. AI 工作流和 Prompt 方案。
5. 长期积累的博客内容。

## 6. 页面需求

### 6.1 首页

首页目标：让访问者在 10 秒内知道站主的定位和网站内容。

核心内容：

1. 一句话个人定位。
2. 简短介绍。
3. 三个核心入口：AI 工具、博客、项目。
4. 精选 AI 工具。
5. 最新博客文章。
6. 精选项目。
7. 联系入口。

首页不做大量营销式文案，避免空泛描述。

### 6.2 AI 工具导航页

AI 工具导航是第一版的核心页面之一。

功能需求：

1. 展示 AI 工具列表。
2. 支持按分类筛选。
3. 支持按标签筛选。
4. 支持关键词搜索。
5. 每个工具卡片展示名称、简介、分类、标签、推荐理由、官网链接。
6. 支持跳转到工具详情页。

建议分类：

1. 写作
2. 编程
3. 搜索
4. 图片
5. 视频
6. 音频
7. 办公
8. 自动化
9. 知识管理
10. Agent

工具详情页内容：

1. 工具名称。
2. 官网链接。
3. 一句话简介。
4. 适合场景。
5. 不适合场景。
6. 核心功能。
7. 个人评价。
8. 替代工具。
9. 标签。
10. 最后更新时间。

### 6.3 博客页

博客用于沉淀长期内容。

功能需求：

1. 展示文章列表。
2. 支持分类筛选。
3. 支持标签筛选。
4. 支持全文搜索。
5. 展示文章标题、摘要、发布日期、阅读时间、标签。
6. 支持 RSS。

建议文章分类：

1. AI 工具
2. 技术开发
3. 项目复盘
4. 学习笔记
5. 产品观察
6. 个人思考

文章详情页内容：

1. 标题。
2. 摘要。
3. 发布日期。
4. 更新时间。
5. 标签。
6. 正文。
7. 相关文章。

### 6.4 项目页

项目页用于展示个人能力和结果。

项目列表需求：

1. 展示项目卡片。
2. 支持按技术栈或类型筛选。
3. 每个项目展示名称、简介、封面图、技术栈、项目状态、详情链接。

项目详情页内容：

1. 项目背景。
2. 项目目标。
3. 技术栈。
4. 核心功能。
5. 实现难点。
6. 解决方案。
7. 项目截图。
8. 在线演示链接。
9. GitHub 或代码仓库链接。
10. 复盘总结。

项目状态建议：

1. 已上线
2. 开发中
3. 原型
4. 已归档

### 6.5 AI 工作流 / Prompt 页

该页面用于强化个人特色，不只展示工具，还展示如何组合工具解决问题。

功能需求：

1. 展示工作流列表。
2. 支持按场景筛选。
3. 展示每个工作流的适用对象、使用工具、步骤和产出。

建议工作流主题：

1. 用 AI 写周报。
2. 用 AI 做竞品分析。
3. 用 AI 辅助写代码。
4. 用 AI 整理知识库。
5. 用 AI 生成文章大纲。
6. 用 AI 做项目需求分析。

工作流详情页内容：

1. 适用场景。
2. 所需工具。
3. 输入材料。
4. 操作步骤。
5. 示例 Prompt。
6. 预期输出。
7. 注意事项。

### 6.6 关于页

Uses 页用于展示常用工具和技术环境。

## 9. 技术栈

### 9.1 V1 推荐技术栈

```txt
Astro + TypeScript + Tailwind CSS + MDX + Pagefind + Cloudflare Pages
```

选择理由：

1. 网站以内容为主，Astro 适合博客、作品集、文档和导航类站点。
2. TypeScript 提升内容模型和组件开发的可靠性。
3. Tailwind CSS 适合快速构建响应式界面。
4. MDX 适合写博客、项目复盘和工具详情。
5. Pagefind 可为静态站提供搜索能力，不需要额外后端。
6. Cloudflare Pages 适合静态部署，成本低，维护简单。

### 9.2 可选增强

```txt
Giscus：GitHub Discussions 驱动的评论系统
Umami：轻量网站数据统计
Iconify 或 Lucide：图标库
Motion One：轻量动效
```

### 9.3 后续动态化技术栈

如果后续需要登录、收藏、评分、投稿、后台管理等能力，可以升级为：

```txt
Next.js + TypeScript + Tailwind CSS + Supabase + Vercel
```

## 11. 设计要求

### 11.1 视觉方向

1. 清爽、专业、有个人气质。
2. 信息密度适中，避免像传统网址导航一样拥挤。
3. 首页优先突出内容入口，而不是大段宣传文案。
4. 工具卡片清晰展示用途、标签和推荐理由。
5. 项目页重视截图和结果呈现。

### 11.2 响应式要求

1. 支持桌面端、平板和手机。
2. 移动端导航清晰可用。
3. 工具筛选在手机端不能占用过多空间。
4. 文章正文在手机端保持良好阅读体验。

### 11.3 可访问性

1. 所有图片提供 alt 文本。
2. 链接和按钮具备清晰的可点击状态。
3. 颜色对比度满足基本阅读要求。
4. 键盘可访问主要导航和筛选控件。

## 12. SEO 要求

V1 需要实现：

1. 每个页面有 title 和 description。
2. 博客、项目、工具详情页生成独立元信息。
3. 生成 sitemap。
4. 支持 RSS。
5. Open Graph 信息。
6. 语义化 HTML 结构。
7. 图片压缩和懒加载。

## 13. 性能要求

1. 首页首屏加载应尽量轻量。
2. 图片使用压缩格式。
3. 非必要交互不引入大型客户端脚本。
4. 静态页面优先生成。
5. 搜索索引在构建时生成。

## 15. 验收标准

V1 完成时应满足：

1. 首页可正常访问，并能进入三大核心模块。
2. AI 工具导航至少包含 20 个工具。
3. 博客至少包含 3 篇文章。
4. 项目页至少包含 3 个项目。
5. 至少有 1 个 AI 工作流页面。
6. 全站在手机端和桌面端可正常浏览。
7. 搜索、筛选或标签导航至少完成一种。
8. 所有核心页面具备基础 SEO 信息。
9. 网站可部署到线上。
10. 内容可以通过 Markdown 或 MDX 持续维护。

## 19. 网站命名方向

可选方向：

1. 个人姓名或昵称作为品牌。
2. “AI + 个人知识库”方向。
3. “实验室 / Lab”方向。
4. “工具箱 / Toolkit”方向。

示例：

1. `YourName Lab`
2. `YourName AI Notes`
3. `AI Toolkit by YourName`
4. `YourName Studio`

最终命名应简短、好记、方便作为域名或社交账号延展。
